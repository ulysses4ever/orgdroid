# M9 — Metadata sheet: edit priority and tags

## Goal
Let the user edit a heading's **priority** (`[#A]/[#B]/[#C]/none`) and its **tags** (`:work:urgent:`). Both fields render as chips today (since M2) but are read-only. M9 makes them editable through a single **Modal Bottom Sheet** opened from a long-press menu item ("Edit metadata"). Changes apply immediately and mark the file dirty; the sheet's "Done" / swipe-down dismissal does not need a confirm step because every individual edit is already committed.

This finally retires the "metadata sheet" label that TODO.md has been pointing at since M5. After M9, every parsed heading field except custom TODO keywords is editable from the UI.

## What we keep from M8
- All M1–M8 behavior unchanged: SAF open/save, outline + collapse + zoom, inline title/notes editing, TODO cycle, indent/outdent, move up/down, conflict detection, recents + auto-open + back-to-recents, search/filter.
- `OrgParser`, `OrgSerializer` (already emits priority + tags from the `Node` fields — verified), `Node`, `NodeId`, `FileIo`, `RecentFiles`, `Search`, `Theme`, `MainActivity` — unchanged.
- `TreeOps` gains two pure helpers (`updatePriority`, `updateTags`). All existing functions unchanged.
- `OutlineState` gains one field (`metadataSheetFor: NodeId?`).
- `OutlineRow` gains one callback (`onEditMetadata`) and a new "Edit metadata" entry in the long-press dropdown.
- Existing chips stay visual-only — tapping a priority/tag chip still routes to the row's `onBeginEdit` (begin title edit). Routing chip taps separately is out of scope and would surprise users who already learned the M5 tap-anywhere-to-edit behavior.

## Out of scope for M9 (don't implement)
- **Custom `#+TODO:` keyword workflows.** Still hardcoded `null → TODO → DONE → null`. Separate milestone.
- **TODO state inside the sheet.** The TODO chip click + "Toggle TODO" menu item already cover this. Adding it to the sheet would be redundant.
- **Priority below `C`.** Org supports `[A-Z]`; the M9 sheet offers `None / A / B / C`. The parser still reads `[#D]` through `[#Z]` as valid; we just don't expose them in the sheet picker. (Existing `priority` values outside `A-C` survive a sheet open + close untouched — see Pitfall 6.)
- **Tag autocomplete.** No "tags used elsewhere in this file" suggestion list. Plain text-entry only.
- **Tag reordering.** Tags display in the order the user added them; no drag-reorder.
- **Multi-select bulk edit.** Sheet operates on one heading at a time.
- **Tap-chip-to-edit.** Tapping a `PriorityChip` or `TagChip` still triggers the row's title edit (because the parent `TitleRow` `combinedClickable` consumes the event). Wrapping each chip in its own `clickable` is fragile — would require routing decisions for users mid-edit. Leave for a future "chip overhaul" milestone.
- **Unicode / `:` / whitespace in tag names.** Tag input validates against `[A-Za-z0-9_@%]+`. Rejected input shows a small "Invalid tag" hint and keeps the field's text so the user can fix it.
- **Confirm on close.** Sheet writes through on every change; no "discard changes?" prompt.
- **Animated chip add/remove.** Plain LazyRow / Row recomposition.
- **Search field that scopes by tag.** M8's free-text search does not become tag-aware in M9.
- **`#+FILETAGS:` inheritance.** Tags shown belong to the heading only; we don't propagate file-level tags.
- **Keyboard shortcuts.**

### 1. `app/src/main/kotlin/dev/orgdroid/org/Tag.kt` (new)

```kotlin
package dev.orgdroid.org

/**
 * Org tag character class: `[A-Za-z0-9_@%]+`. Mirrors the heading-line regex
 * used by [OrgParser], minus the `:` and `#` separators which are structural.
 * A tag is valid iff it is non-empty and every character is in the class.
 */
fun isValidTag(s: String): Boolean =
    s.isNotEmpty() && s.all { it.isLetterOrDigit() || it == '_' || it == '@' || it == '%' }
```

**5b. Add five new methods.** Group them after `closeSearch` / `setSearchQuery`:

```kotlin
fun openMetadata(id: NodeId) {
    val s = commitEditInternal(_state.value)
    val root = s.root ?: return
    if (TreeOps.findNode(root, id) == null) return
    _state.value = s.copy(metadataSheetFor = id)
}

fun closeMetadata() {
    _state.value = _state.value.copy(metadataSheetFor = null)
}

fun setPriority(id: NodeId, priority: Char?) {
    val s = _state.value
    val root = s.root ?: return
    val node = TreeOps.findNode(root, id) ?: return
    if (node.priority == priority) return
    _state.value = s.copy(root = TreeOps.updatePriority(root, id, priority), dirty = true)
}

fun addTag(id: NodeId, tag: String) {
    val s = _state.value
    val root = s.root ?: return
    val node = TreeOps.findNode(root, id) ?: return
    if (tag in node.tags) return
    _state.value = s.copy(
        root = TreeOps.updateTags(root, id, node.tags + tag),
        dirty = true,
    )
}

fun removeTag(id: NodeId, tag: String) {
    val s = _state.value
    val root = s.root ?: return
    val node = TreeOps.findNode(root, id) ?: return
    if (tag !in node.tags) return
    _state.value = s.copy(
        root = TreeOps.updateTags(root, id, node.tags - tag),
        dirty = true,
    )
}
```

Notes:
- `openMetadata` commits any pending title/notes edit first — same pattern as `openSearch`. Without this, a half-typed title is stranded when the sheet steals focus.
- `addTag` does NOT validate; validation happens in the UI before this call. The ViewModel trusts its caller (only `MetadataSheet` invokes it).
- `setPriority` / `addTag` / `removeTag` no-op when the value is already what the user requested. Avoids spurious dirty flips and recompositions.
- None of these mutate `metadataSheetFor`; the sheet stays open across edits.

    onAddTag: (String) -> Unit,
    onRemoveTag: (String) -> Unit,
    onDismiss: () -> Unit,

    val sheetState = rememberModalBottomSheetState(skipPartiallyExpanded = true)
    var tagInput by rememberSaveable { mutableStateOf("") }
    var tagError by rememberSaveable { mutableStateOf<String?>(null) }

    ModalBottomSheet(onDismissRequest = onDismiss, sheetState = sheetState) {
        Column(Modifier.padding(horizontal = 16.dp, vertical = 8.dp)) {

                color = MaterialTheme.colorScheme.onSurfaceVariant,

                PriorityRadio("None", node.priority == null) { onSetPriority(null) }
                PriorityRadio("A", node.priority == 'A') { onSetPriority('A') }
                PriorityRadio("B", node.priority == 'B') { onSetPriority('B') }
                PriorityRadio("C", node.priority == 'C') { onSetPriority('C') }

            Text(
                "Tags",
                style = MaterialTheme.typography.titleSmall,
                color = MaterialTheme.colorScheme.onSurfaceVariant,
                modifier = Modifier.padding(bottom = 8.dp),
            )

                Text(
                    "No tags yet",
                    style = MaterialTheme.typography.bodyMedium,
                    color = MaterialTheme.colorScheme.onSurfaceVariant,
                    modifier = Modifier.padding(bottom = 8.dp),
                )
            } else {
                FlowRow(

private fun commitTag(
    raw: String,
    existing: List<String>,
    onAdd: (String) -> Unit,
    after: (String?) -> Unit,
) {
    val tag = raw.trim()
    when {
        tag.isEmpty() -> after(null) // silently ignore — don't clear, don't add
        !isValidTag(tag) -> after("Invalid tag: use letters, digits, _ @ %")
        tag in existing -> after(null) // duplicate — clear input via caller, no error
        else -> { onAdd(tag); after(null) }
    }
}

## Pitfalls (read before coding)

1. **`FlowRow` requires `ExperimentalLayoutApi`.** Forgetting the opt-in produces an opaque compile error. Add `@OptIn(ExperimentalLayoutApi::class)` at the call site or the file.
2. **`ModalBottomSheet` requires `ExperimentalMaterial3Api`.** The `OutlineScreen` already opts in for `TopAppBar`. The `MetadataSheet` composable needs its own `@OptIn(ExperimentalMaterial3Api::class)` because it's a separate composable function.
3. **Don't capture `node` at sheet-open time.** Re-resolve `metaNode = TreeOps.findNode(state.root, metaId)` on every render. Otherwise the chips list won't update after `addTag` / `removeTag` until the sheet is reopened.
4. **Sheet dismisses on configuration change unless guarded.** `metadataSheetFor` lives in the ViewModel, so rotation preserves it; the sheet recomposes against the new node. But `sheetState` is created fresh per composition — it'll snap to the expanded position again. Acceptable.
5. **`onSetPriority(null)` must be reachable.** The "None" radio is the only way to remove a priority. Don't accidentally drop the `null` branch in the radio list.
6. **Heading with `[#D]` priority.** Org allows `[#A-Z]`; we expose only `A-C`. When the current value is outside `A-C`, NONE of the radios should be `selected = true`. The user can pick a different priority or close the sheet; the original `[#D]` survives a sheet open + close as long as no radio was clicked. Test this explicitly.
7. **Tags ARE case-sensitive in dedupe.** `addTag` checks `tag in node.tags`. `Work` and `work` coexist. Org-mode itself treats tags as case-sensitive, so this matches upstream behavior.
8. **Search (M8) does NOT index tags.** `Search.visibleIds` only inspects `node.title` and `node.notes`. After M9, a heading with `:urgent:` won't match the query "urgent" unless the title or notes also contain it. This is a known gap; defer "search tags too" to a later milestone.
9. **`updateTags` with empty list must clear the trailing `:tag:` block in serialization.** Confirmed: `OrgSerializer.reconstructHeading` guards on `node.tags.isNotEmpty()` before appending the block. The explicit test `updateTagsToEmptyClearsHeadingTagBlock` in §4 locks this.
10. **`commitTag` separates "invalid" from "silent ignore".** Empty input on Done/Add should not flash an error (the user might be hitting Done to dismiss the keyboard). Duplicate tags also silently no-op. Only character-class violations show "Invalid tag".
11. **`PriorityRadio` makes both the radio AND the label clickable.** The `Row` has its own `.clickable` so tapping the text label selects the radio. Without that, only the tiny circle hitbox works.
12. **The dropdown menu now has 10 items.** TODO.md flagged 9 after M6; M9 adds "Edit metadata" → 10. The "grouping or bottom sheet variant" deferred-polish note still applies.
13. **Trimming the tag input.** `commitTag` calls `raw.trim()` so a user typing `" work "` produces `work`. Don't trim *during* `onValueChange` — that mid-type prevention is jarring for IMEs that insert a trailing space before the next composition.
14. **Don't add `metadataSheetFor` to the dependency list of the `items` `derivedStateOf`.** The flatten output doesn't depend on the sheet being open; opening the sheet must not invalidate the LazyColumn.
15. **Don't dismiss the sheet from inside `setPriority` / `addTag` / `removeTag`.** Users expect to make multiple edits in one sheet visit. Only `closeMetadata` (called by `onDismissRequest`) closes the sheet.
16. **`rememberSaveable` for `tagError`.** It's a `String?` which `Saveable` handles natively. No custom saver needed.
17. **The "Add" button has no `enabled` gate.** Empty input is a silent no-op; non-empty invalid input shows an error. Disabling the button would prevent the user from triggering the error message and learning the rules.
18. **`OutlineRow` parameter count.** Adding `onEditMetadata` brings it to 22. Already flagged in TODO.md as needing a `OutlineRowCallbacks` bundle. M9 does NOT introduce the bundle — that's a separate refactoring milestone. Stay consistent with M8.
19. **No on-device persistence test required.** The sheet writes through `OutlineViewModel.setPriority/addTag/removeTag`, which mark dirty. The user must still tap **Save**. The "save → reopen → metadata persists" acceptance criterion is the round-trip check.
20. **`IconButton(modifier = Modifier.size(24.dp))` does not shrink the ripple.** Material 3 enforces a 40dp minimum touch target on `IconButton`; the override resizes the icon but the ripple still expands to 40dp on press. The visual is acceptable for M9; if a smaller hit target is wanted later, replace with `Box(Modifier.size(24.dp).clickable { … }) { Icon(…) }`.
21. **`closeFile()` and `delete()` carry one-line clears for `metadataSheetFor`** (see §5c). Without them, the sheet sits behind the close-confirm dialog or points at a deleted node id. The `findNode` null-guard in §6c keeps the latter from crashing, but the stale state lingers until the next user action.

## What M9 explicitly does NOT do

- TODO state editing from the sheet (chip click + menu cover it).
- Custom `#+TODO:` keyword workflows.
- Priority `D`-`Z` selection (parser preserves them; sheet only offers A/B/C/None).
- Tag autocomplete or suggestions.
- Tag reordering / drag handles.
- Tap-on-chip-to-edit (chips remain visual only).
- Multi-heading bulk metadata edit.
- Unicode tags or tags containing `:`/space/`#`.
- Search-by-tag.
- File-level `#+FILETAGS:` inheritance.
- Sheet-level "discard changes" confirmation (every edit is already committed).
- Keyboard shortcuts.
- Animated chip add/remove transitions.
- A refactor of `OutlineState` or `OutlineRow` parameter lists.
