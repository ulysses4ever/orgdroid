# M8 — Search / filter: hide non-matching headings

## Goal
Give the user a way to find a heading by text without scrolling. Tap a Search icon → a search field appears below the top bar. As the user types, the outline filters to only rows whose title or notes contain the query (case-insensitive substring). Ancestors of matches stay visible so each match has context. Tap Close (or the back-arrow on the search bar) to clear the query and return to the full outline.

This is filter mode (Workflowy-style), not find mode. Non-matching rows disappear; there is no "next match" navigation. Filter mode is the right primitive for an outliner because the whole point is the outline view itself — the user wants to *see* the matches together, not jump between them.

## What we keep from M7
- All M1–M7 behavior unchanged: SAF open/save, outline + collapse + zoom, inline title/notes editing, TODO cycle, indent/outdent, move up/down, conflict detection, recents + auto-open + back-to-recents.
- `OrgParser`, `OrgSerializer`, `Node`, `NodeId`, `TreeOps`, `FileIo`, `RecentFiles`, `Theme`, `MainActivity` — unchanged.
- Existing `OutlineState` fields stay; we add two (`searchActive: Boolean`, `searchQuery: String`).
- All M3 conflict handling, M7 recents, and the Close button work the same way during search.

## Out of scope for M8 (don't implement)
- **Find mode** (highlight match in place, n/p navigation). Filter mode only.
- **Regex search.** Plain substring match.
- **Case-sensitive search toggle.** Always case-insensitive.
- **Find-and-replace.**
- **Search across multiple files** (recents). Active file only.
- **Persisted search history** ("recently searched" suggestions).
- **Match counter** ("3 matches"). The filtered list is the count; no separate UI.
- **Highlight the matching substring inside the title text.** Just filter — no `AnnotatedString` styling. Defer to M9.
- **Search inside the synthetic root's notes** (the file header before the first heading). The root is never an emitted row; treating it as searchable would force "everything visible" semantics. Skip it.
- **Debounce / async**. The filter runs synchronously on every keystroke. Files under ~20k headings should be fine.
- **Auto-scroll to first match.** Defer.
- **Keyboard shortcut** (e.g., Ctrl-F). No hardware-keyboard story yet.
- **Search bar in the recents-list empty state.** Search only applies when a file is open.
- **Saved searches / bookmark queries.**

## File contents

### 1. `app/src/main/kotlin/dev/orgdroid/org/Search.kt` (new)

```kotlin
package dev.orgdroid.org

/**
 * Outline-aware substring search. Returns the set of [NodeId]s that should
 * be visible for [query]:
 *
 *   - A node is a "match" iff its title or any of its notes lines contains
 *     [query] (case-insensitive).
 *   - A node is "visible" iff it is a match OR any of its descendants is
 *     visible. This keeps ancestor context around each match.
 *
 * Returns the empty set for an empty query; callers should treat that as
 * "no filter, show everything" rather than "filter to nothing".
 *
 * The synthetic root's own title/notes are NOT searched: the root has no
 * row and treating it as searchable would force every descendant visible.
 */
object Search {
    fun visibleIds(root: Node, query: String): Set<NodeId> {
        if (query.isEmpty()) return emptySet()
        val needle = query.lowercase()
        val visible = mutableSetOf<NodeId>()
        // Root is never a "match"; only its descendants contribute.
        for (c in root.children) {
            collect(c, needle, visible)
        }
        if (visible.isNotEmpty()) visible.add(root.id)
        return visible
    }

    private fun collect(node: Node, needle: String, out: MutableSet<NodeId>): Boolean {
        val selfMatch = node.title.lowercase().contains(needle) ||
            node.notes.any { it.lowercase().contains(needle) }
        var anyChildVisible = false
        for (c in node.children) {
            if (collect(c, needle, out)) anyChildVisible = true
        }
        val nodeVisible = selfMatch || anyChildVisible
        if (nodeVisible) out.add(node.id)
        return nodeVisible
    }
}
```

Notes for the implementer:
- The function is pure and takes no Context. Lives in `org/` next to the parser, not in `ui/`.
- `String.lowercase()` does Unicode-aware folding by default. Good enough for M8; per-locale folding can be added later if it bites.
- `out.add(root.id)` at the end is a small convenience: if any descendant is visible, mark the root too. The UI doesn't render the synthetic root, but this keeps the set self-consistent (every ancestor chain is captured).
- O(N) over the tree per call. For a 10k-heading file this is ~ms; per-keystroke is fine.

import org.junit.Assert.assertEquals
import org.junit.Assert.assertFalse

class SearchTest {

            * Project Alpha

            ** Notes

            * Meeting notes
        """.trimIndent() + "\n"

    @Test fun matchOnTitle() {

        val visible = Search.visibleIds(root, "Beta")

        assertTrue("Project Beta" in byTitle)

        assertFalse("Meeting notes" in byTitle)

        val root = tree()

        val visible = Search.visibleIds(root, "done with")
        val byTitle = visible.mapNotNull { TreeOps.findNode(root, it)?.title }.toSet()
        assertTrue("Notes" in byTitle)
        assertTrue("Project Alpha" in byTitle) // ancestor preserved
        assertFalse("Project Beta" in byTitle)
    }

    @Test fun caseInsensitiveSubstring() {
        val root = tree()
        val a = Search.visibleIds(root, "ALPHA")
        val b = Search.visibleIds(root, "alpha")
        assertEquals(a, b)
    }

    @Test fun noMatchReturnsEmpty() {
        val root = tree()
        assertTrue(Search.visibleIds(root, "zzz-no-such").isEmpty())
    }

    @Test fun rootIncludedWhenAnyDescendantMatches() {
        val root = tree()
        val visible = Search.visibleIds(root, "Project")
        assertTrue(root.id in visible)

    @Test fun descendantsOfMatchAreNotAutoIncluded() {
        // Matching "Project Alpha" should NOT auto-include its children.
        val root = tree()
        val visible = Search.visibleIds(root, "Project Alpha")
        val titles = visible.mapNotNull { TreeOps.findNode(root, it)?.title }.toSet()
        assertTrue("Project Alpha" in titles)
        assertFalse("Design doc" in titles)
        assertFalse("Notes" in titles)

**4b. Extend `flatten`** — signature gains `visibleIds: Set<NodeId>`. When non-empty, the function ignores `collapsed` and skips any node not in the set:

```kotlin
private fun flatten(
    root: Node?,
    collapsed: Set<NodeId>,
    focusedRoot: NodeId?,
    visibleIds: Set<NodeId>,
): List<RenderItem> {
    if (root == null) return emptyList()
    val startNode = if (focusedRoot != null) TreeOps.findNode(root, focusedRoot) else root
    if (startNode == null) return emptyList()
    val filtering = visibleIds.isNotEmpty()

    val out = mutableListOf<RenderItem>()
    fun visit(parent: Node, node: Node, indexInParent: Int, depth: Int) {
        if (filtering && node.id !in visibleIds) return
        val isCollapsed = node.id in collapsed
        if (node.id != startNode.id) {
            out.add(
                RenderItem(
                    node = node,
                    depth = depth,
                    hasChildren = node.children.isNotEmpty(),
                    isCollapsed = if (filtering) false else isCollapsed,
                    canIndent = indexInParent > 0,
                    canOutdent = parent.id != startNode.id,
                    canMoveUp = indexInParent > 0,
                    canMoveDown = indexInParent < parent.children.size - 1,
                )
            )
        }
        val expandChildren = if (filtering) true else !isCollapsed
        if (expandChildren) {
            val childDepth = if (node.id == startNode.id) 0 else depth + 1
            for ((i, c) in node.children.withIndex()) visit(node, c, i, childDepth)
        }
    }
    visit(startNode, startNode, -1, 0)
    return out
}
```

Two key behaviors:
- `isCollapsed = if (filtering) false else isCollapsed` — bullet drawing during search shows "expanded" even on collapsed nodes, so the user isn't confused by a "collapsed" bullet whose children are still rendered below.
- `expandChildren = if (filtering) true else !isCollapsed` — during search, walk through collapsed nodes too. After closing search, the original collapse state is back in effect.

private fun SearchBarRow(
    query: String,
    onChange: (String) -> Unit,
    onClose: () -> Unit,

            Icon(Icons.Filled.ArrowBack, contentDescription = "Close search")

        Box(Modifier.weight(1f)) {
            if (query.isEmpty()) {
                Text(
                    "Search…",
                    style = MaterialTheme.typography.bodyLarge,
                    color = MaterialTheme.colorScheme.onSurfaceVariant,
                )
            }
            BasicTextField(
                value = query,
                onValueChange = onChange,
                modifier = Modifier.fillMaxWidth().focusRequester(focusRequester),
                singleLine = true,
                textStyle = MaterialTheme.typography.bodyLarge.copy(
                    color = MaterialTheme.colorScheme.onSurface,
                ),
                cursorBrush = SolidColor(MaterialTheme.colorScheme.primary),
            )
        }
        if (query.isNotEmpty()) {
            IconButton(onClick = { onChange("") }) {
                Icon(Icons.Filled.Close, contentDescription = "Clear search")
            }
        }
    }
}

## Pitfalls (read before coding)

1. **`Search.visibleIds` returns `emptySet()` for an empty query.** Callers MUST treat that as "no filter, show everything", not "filter to nothing". This is why `flatten` checks `visibleIds.isNotEmpty()` for `filtering`, not `visibleIds != null` or some other sentinel.
2. **Ignore `collapsed` during search.** A match buried inside a collapsed subtree would otherwise be invisible — defeating the purpose. The `flatten` change is two lines (`isCollapsed = if (filtering) false else …` and `expandChildren = if (filtering) true else !isCollapsed`). Don't forget either one.
3. **Don't search the synthetic root's own title or notes.** The root has no row and treating it as "matching" would mark every descendant visible. `Search.kt` skips it explicitly: it iterates `root.children`, never inspects `root.title` / `root.notes`.
4. **`openSearch()` must commit any pending edit FIRST.** Otherwise a half-typed title disappears into the void when the search field grabs focus. The pattern is `val s = commitEditInternal(_state.value); _state.value = s.copy(searchActive = true)`. Same shape as `closeFile` and the M4 mutators.
5. **`setSearchQuery` guards on `searchActive`.** Without this guard, a stale `onValueChange` callback (Compose can fire one for a frame after the SearchBarRow leaves composition) can mutate state and re-open the search visually. The guard makes `closeSearch()` truly idempotent.
6. **Two `derivedStateOf` blocks**, not one. `visibleIds` keys on `(root, query)`; `items` keys on `(root, collapsed, focusedRoot, visibleIds)`. Folding them together would re-compute the search whenever `collapsed` flips.
7. **`@Stable` for `Set<NodeId>`.** Compose treats `Set` as unstable by default, but since `visibleIds` is captured inside `derivedStateOf` and its identity changes on each re-derivation, the recomposition cost is bounded by the number of items, not by stability. No `@Stable` annotation needed.
8. **The search field's focus requester runs once.** `LaunchedEffect(Unit)` is intentionally keyed on `Unit`, not on `query`. Re-running the focus request on every keystroke would scroll the cursor or briefly drop focus. The field stays focused as long as the row is composed.
9. **Placeholder must NOT be inside `BasicTextField`'s `decorationBox`.** That parameter is a slot for the entire decoration around the inner text, not for an "if empty show this" condition. Using a sibling `Text` in a `Box` is simpler.
10. **`Icons.Filled.Search` is part of `material-icons-core`** which is on the classpath (we already use `Icons.Filled.Add`, `Close`, `ArrowBack`). No new dep.
11. **Empty-state branch order in the `when` block** matters. The "No matches" branch must be inside the existing `else ->` (the "we have a root, no loading, no error" branch). Adding it as a top-level branch above the existing checks would cause it to fire whenever `searchQuery` is non-empty regardless of file load state.
12. **`combinedClickable` on `RecentRow`** stays unchanged. Search has no interaction with the recents list — search bar only shows when a file is open.
13. **During search, the bullet still toggles collapse.** Tapping a bullet during search changes `state.collapsed`, but since `flatten` ignores `collapsed` while filtering, the visual result doesn't change until search closes. That's slightly confusing but acceptable: closing search reveals the new collapse state.
14. **`Search.kt` lives in `org/`, not `ui/`.** It is pure tree logic; keep it next to `TreeOps`, `OrgParser`, etc. for symmetry.
15. **`Set<NodeId>` for visibility, not `Set<Long>`.** `NodeId` is a `@JvmInline value class` so the box cost is zero at runtime. Use the typed wrapper everywhere it would appear in API surfaces.
16. **Don't add an "open search" affordance from the recents-list empty state.** Search only applies to an open file. The Search button in the top bar is gated on `state.root != null`.
17. **Save / conflict / close / FAB interactions during search**: all unchanged. The mutating ops update the tree, the next render's `visibleIds` derivation reflects the new content. No special-casing needed.
18. **`SearchBarRow` does NOT call `onChange("")` on close.** `closeSearch()` in the ViewModel resets `searchQuery` to `""`; the row's `onClose` callback only flips `searchActive`. Calling `onChange("")` from the row would cause an extra recomposition with `searchActive = true && query = ""` for a frame.
