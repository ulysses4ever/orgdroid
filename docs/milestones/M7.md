# M7 — Recent files: persist URIs, auto-open, back-to-recents

## Goal
Eliminate the cold-start friction of having to navigate SAF every time the app opens. Persist a small list of recently-opened URIs. On launch, auto-open the most recent. From inside an open file, allow the user to "close" back to a recents list. Long-press a recent to remove it.

After M7, the daily flow for a single-file user is "tap app icon → see your file". For multi-file users, "tap app icon → most recent opens → tap close → tap the file you want."

## UX summary
- **Cold start, empty recents**: existing "No file open. Tap Open." text. SAF picker via the existing top-bar Open button.
- **Cold start, non-empty recents**: ViewModel `init` reads the persisted list and triggers `open(mostRecent.uri)` synchronously after a short load. On success, the outline appears as today. On failure (URI revoked, file deleted), the user is dropped into the recents-list empty state with an error banner above the list.
- **In an open file**: a new "Close" icon button in the top bar (between the title and Open). Tapping it returns to the recents-list empty state. If the file is dirty, a confirmation dialog asks "Discard unsaved changes?" with Discard / Cancel.
- **On the recents-list empty state**: each row is the file name (no timestamp in M7 — keep the list lean). Tap to open. Long-press → menu with a single item, "Remove from recents".
- **Capacity**: 10 most-recently-opened URIs. Re-opening an existing entry bumps it to the top.

## What we keep from M6
- M1–M6 behavior unchanged: SAF open, save with conflict detection, outline + collapse, inline title/notes editing, TODO cycle, indent/outdent, zoom in/out, move up/down.
- `OrgParser`, `OrgSerializer`, `Node`, `NodeId`, `TreeOps`, `FileIo`, `Theme`, `MainActivity` — all unchanged.
- Existing `OutlineState` fields stay; we add one (`recents: List<RecentFile>`) and one (`closePending: Boolean`).

## Out of scope for M7 (don't implement)
- A "Recents" screen reachable from anywhere besides the empty state and the close button.
- File previews / thumbnails / heading-count badges next to each recent.
- A relative-time display ("opened 2 hours ago"). Filename only in M7.
- Pinning / favoriting recents.
- A "rename file" action.
- Detection of file rename via content provider (`OpenableColumns.DISPLAY_NAME`). We use `Uri.lastPathSegment` for display, same as today.
- Persisting per-file UI state (collapse set, focused root, scroll position).
- DataStore / kotlinx.serialization / room. SharedPreferences only.
- A setting to disable auto-open. It is always on in M7. Adding a settings screen is M8+.
- Sync across devices.
- Re-issuing SAF permission requests when a URI's grant has been revoked. Failure just removes the entry from recents (or the user removes manually).
- Bulk "clear all recents" action. Single-entry removal only.

### 1. `app/src/main/kotlin/dev/orgdroid/recents/RecentFiles.kt` (new)

Contains three things:
- `RecentFile` data class (the value type stored per entry)
- `RecentFilesCodec` object (pure encode/decode, unit-testable)
- `RecentFilesStore` class (SharedPreferences-backed; needs Android Context)

```kotlin
package dev.orgdroid.recents

import android.content.Context

data class RecentFile(
    val uri: String,           // android.net.Uri.toString() — round-trips via Uri.parse
    val displayName: String,   // for UI; derived from uri.lastPathSegment at record time
    val lastOpenedAt: Long,    // System.currentTimeMillis()
)

/**
 * Pure encode/decode. One entry per line, tab-delimited:
 *
 *   <timestamp>\t<escaped-displayName>\t<uri>
 *
 * URIs are RFC-encoded by Android and contain no tabs or newlines, so they
 * are stored raw. Timestamps are decimal Longs. Only [RecentFile.displayName]
 * needs escaping: `\` → `\\`, `\n` → `\n` (literal), `\t` → `\t` (literal).
 *
 * Decode is forgiving: blank lines and malformed lines (wrong field count,
 * non-numeric timestamp) are skipped silently. This lets us evolve the
 * format later without crashing on stale data.
 */
object RecentFilesCodec {
    fun encode(items: List<RecentFile>): String {
        if (items.isEmpty()) return ""
        val sb = StringBuilder()
        for (item in items) {
            sb.append(item.lastOpenedAt)
            sb.append('\t')
            sb.append(escape(item.displayName))
            sb.append('\t')
            sb.append(item.uri)
            sb.append('\n')
        }
        return sb.toString()
    }

    fun decode(text: String): List<RecentFile> {
        if (text.isEmpty()) return emptyList()
        val out = mutableListOf<RecentFile>()
        for (line in text.split('\n')) {
            if (line.isEmpty()) continue
            val parts = line.split('\t', limit = 3)
            if (parts.size != 3) continue
            val ts = parts[0].toLongOrNull() ?: continue
            out.add(
                RecentFile(
                    uri = parts[2],
                    displayName = unescape(parts[1]),
                    lastOpenedAt = ts,
                )
            )
        }
        return out
    }

    private fun escape(s: String): String =
        s.replace("\\", "\\\\").replace("\n", "\\n").replace("\t", "\\t")

    private fun unescape(s: String): String {
        val sb = StringBuilder()
        var i = 0
        while (i < s.length) {
            val c = s[i]
            if (c == '\\' && i + 1 < s.length) {
                when (s[i + 1]) {
                    'n' -> sb.append('\n')
                    't' -> sb.append('\t')
                    '\\' -> sb.append('\\')
                    else -> {
                        sb.append(c); sb.append(s[i + 1])
                    }
                }
                i += 2
            } else {
                sb.append(c)
                i += 1
            }
        }
        return sb.toString()
    }
}

/**
 * Thin SharedPreferences wrapper. All operations are synchronous; the read
 * is a single map lookup and the write uses [apply] (async commit). Cap at
 * [CAPACITY] entries; oldest dropped on overflow.
 */
class RecentFilesStore(context: Context) {
    private val prefs = context.applicationContext
        .getSharedPreferences(PREFS_NAME, Context.MODE_PRIVATE)

    fun load(): List<RecentFile> {
        val text = prefs.getString(KEY_LIST, "") ?: ""
        return RecentFilesCodec.decode(text)
    }

    /** Add or bump [item] to the top. Returns the new list. */
    fun add(item: RecentFile): List<RecentFile> {
        val current = load().filter { it.uri != item.uri }
        val updated = (listOf(item) + current).take(CAPACITY)
        save(updated)
        return updated
    }

    /** Remove entry with [uri]. Returns the new list. No-op if not present. */
    fun remove(uri: String): List<RecentFile> {
        val updated = load().filter { it.uri != uri }
        save(updated)
        return updated
    }

    private fun save(items: List<RecentFile>) {
        prefs.edit().putString(KEY_LIST, RecentFilesCodec.encode(items)).apply()
    }

    companion object {
        private const val PREFS_NAME = "orgdroid_recents"
        private const val KEY_LIST = "list"
        const val CAPACITY = 10
    }
}
```

Notes for the implementer:
- `RecentFilesCodec` is a pure `object`. Don't put a `Context` field on it — that breaks unit testability.
- `RecentFilesStore` is the only place that touches SharedPreferences. Don't sprinkle `getSharedPreferences` calls through the ViewModel.
- The `applicationContext` indirection guards against accidentally holding an Activity reference.
- `apply()` (not `commit()`) on edits — non-blocking, write-behind. Safe given we always re-load before mutating.
- The codec's `decode` is intentionally forgiving (skip malformed lines). This is forward-compat: a future version that adds a fourth field can keep reading old data, and an old version that finds a new format won't crash.

### 2. `app/src/test/kotlin/dev/orgdroid/recents/RecentFilesCodecTest.kt` (new)

```kotlin
package dev.orgdroid.recents

import org.junit.Assert.assertEquals
import org.junit.Assert.assertTrue
import org.junit.Test

class RecentFilesCodecTest {

    @Test fun emptyListEncodesToEmptyString() {
        assertEquals("", RecentFilesCodec.encode(emptyList()))
    }

    @Test fun emptyStringDecodesToEmptyList() {
        assertTrue(RecentFilesCodec.decode("").isEmpty())
    }

    @Test fun singleEntryRoundTrips() {
        val item = RecentFile(
            uri = "content://com.example/doc/123",
            displayName = "test.org",
            lastOpenedAt = 1_700_000_000_000L,
        )
        val encoded = RecentFilesCodec.encode(listOf(item))
        val decoded = RecentFilesCodec.decode(encoded)
        assertEquals(listOf(item), decoded)
    }

    @Test fun multipleEntriesPreserveOrder() {
        val a = RecentFile("uri-a", "a.org", 1L)
        val b = RecentFile("uri-b", "b.org", 2L)
        val c = RecentFile("uri-c", "c.org", 3L)
        val items = listOf(a, b, c)
        val roundTripped = RecentFilesCodec.decode(RecentFilesCodec.encode(items))
        assertEquals(items, roundTripped)
    }

    @Test fun displayNameWithSpecialCharsRoundTrips() {
        val item = RecentFile(
            uri = "u",
            displayName = "weird\tname\nwith\\escapes.org",
            lastOpenedAt = 42L,
        )
        val decoded = RecentFilesCodec.decode(RecentFilesCodec.encode(listOf(item)))
        assertEquals(item, decoded.single())
    }

    @Test fun malformedLinesAreSkipped() {
        val good = "1700000000000\tone.org\turi-1\n"
        val bad = "not\ta\tvalid\tline\n"   // 4 fields
        val alsoBad = "nope\n"              // 1 field
        val nonNumericTs = "abc\tname\turi\n"
        val combined = good + bad + alsoBad + nonNumericTs
        val decoded = RecentFilesCodec.decode(combined)
        assertEquals(1, decoded.size)
        assertEquals("one.org", decoded[0].displayName)
    }

    @Test fun blankLinesAreSkipped() {
        val text = "1\ta\turi-1\n\n2\tb\turi-2\n"
        val decoded = RecentFilesCodec.decode(text)
        assertEquals(2, decoded.size)
    }
}
```

**3d. Update `open()`** to preserve `recents` across the state reset, swallow `SecurityException` from `persistPermission`, and call `recordAccess` on success:

```kotlin
fun open(uri: Uri) {
    val resolver = getApplication<Application>().contentResolver
    try {
        FileIo.persistPermission(resolver, uri)
    } catch (_: SecurityException) {
        // No persistable grant; the subsequent read will fail with a clear error.
    }
    val name = uri.lastPathSegment?.substringAfterLast('/')
    val keepRecents = _state.value.recents
    _state.value = OutlineState(
        uri = uri,
        fileName = name,
        loading = true,
        recents = keepRecents,
    )
    viewModelScope.launch {
        try {
            val text = FileIo.read(resolver, uri)
            val root = withContext(Dispatchers.Default) { OrgParser.parse(text) }
            nextNodeIdValue = TreeOps.maxNodeIdValue(root) + 1L
            val updatedRecents = recordAccess(uri, name)
            _state.value = OutlineState(
                uri = uri,
                fileName = name,
                root = root,
                originalText = text,
                recents = updatedRecents,
            )
        } catch (t: Throwable) {
            _state.value = _state.value.copy(loading = false, error = t.message)
        }
    }
}

private fun recordAccess(uri: Uri, name: String?): List<RecentFile> {
    val display = name ?: uri.lastPathSegment ?: uri.toString()
    return store.add(
        RecentFile(
            uri = uri.toString(),
            displayName = display,
            lastOpenedAt = System.currentTimeMillis(),
        )
    )
}
```

Key changes vs M6's `open()`:
1. The `OutlineState(uri = uri, fileName = name, loading = true)` call now includes `recents = keepRecents`. Without this, recents are wiped on every open.
2. The success branch's `OutlineState(...)` likewise carries `recents = updatedRecents`.
3. `recordAccess` runs in the success branch only — failed opens do not bump the timestamp.
4. `persistPermission` is wrapped in a try/catch. Today it can throw `SecurityException` if the URI grant was revoked; that's the auto-open-of-a-dead-URI path.

    val committed = commitEditInternal(_state.value)

## Pitfalls (read before coding)

1. **`recents` must survive `open()`'s state reset.** The existing `open()` does `_state.value = OutlineState(uri = uri, fileName = name, loading = true)` — a fresh `OutlineState`. If you forget to thread `recents = keepRecents`, the list wipes on every open and the UI flickers from "Recent files" to "No file open" between SAF-pick and parse. Catch this with manual testing.
2. **Don't record access on failed reads.** The `recordAccess` call sits inside the success branch of `open()`. If you put it before the `viewModelScope.launch`, a dead URI will bump its own timestamp every launch and never age out.
3. **`persistPermission` can throw `SecurityException` on auto-open.** Wrap in try/catch in `open()`. Not in `FileIo` — M1's contract is "throws on failure" and other callers may want that.
4. **`init` runs once per ViewModel lifecycle, not once per Activity onCreate.** Auto-open happens once on cold start (correct). It does NOT fire on rotation (correct — rotation preserves the ViewModel). Test rotation explicitly.
5. **Synchronous SharedPreferences read in `init`.** The lookup is a single map access; ~µs. Don't async this.
6. **`closeFile` must commit pending edits first** before checking `dirty`. Otherwise a half-typed title that hasn't been committed still counts as not-dirty, and you close without warning. Pattern: `val s = commitEditInternal(_state.value); _state.value = s; if (s.dirty) …`.
7. **The Close button should appear even during loading.** Use `state.root != null || state.uri != null` for the visibility check. Otherwise a slow auto-open of a giant file traps the user in the spinner with no escape.
8. **Capacity of 10 lives in `RecentFilesStore.CAPACITY`.** Don't duplicate this in the ViewModel.
9. **`Uri.parse` of a stored URI string can throw** (defensively). Wrap the `init` block's `open(Uri.parse(...))` call in a try/catch. We do — see code above.
10. **Don't use `JSONObject`/`JSONArray` for the codec.** They're on the Android classpath but not the unit-test classpath. Tests would either need Robolectric (overkill) or `returnDefaultValues=true` (silent breakage). Hand-rolled escape format is small enough.
11. **The codec must tolerate forward-incompatible data.** If a future version stores a 4th field, today's decoder skips those lines (good). If we ship a new field as a new line type, today's decoder skips that too. Make sure `decode` never throws on bad input — only skips.
12. **Don't `commit()` on SharedPreferences — use `apply()`.** `commit()` is synchronous-blocking; `apply()` is fire-and-forget. We always re-`load()` before mutating so write ordering is not load-bearing.
13. **`combinedClickable` on `RecentRow` requires `ExperimentalFoundationApi`.** Add `@OptIn(ExperimentalFoundationApi::class)` to `EmptyState` and `RecentRow`. The existing file already opts in for `TitleRow` — add the annotation per new function, not at file level (Compose convention is per-symbol).
14. **The `DropdownMenu` inside `RecentRow` has the same scroll-out fragility** as `OutlineRow`'s menu (already flagged in TODO.md). Same mitigation: the menu scrim blocks scroll while open. Don't add to TODO.md unless the issue actually bites.
15. **`store.add(item)` returns the new list synchronously.** Do not call `store.load()` after to "refresh" — `add()` already gives you the post-mutation list.
16. **The dirty-confirm dialog's `onDismissRequest`** should map to **cancel**, not discard. Tapping outside the dialog is the user expressing "I changed my mind", not "yes burn my data".
17. **`closeFile()` keeps `recents` but drops everything else.** Don't accidentally re-init `nextNodeIdValue` here; a subsequent open will reset it. Don't accidentally re-init `recents` here; that would wipe the user's list every close.
18. **No file-system probing in `recordAccess`.** Don't try to query `OpenableColumns.DISPLAY_NAME` or check file existence. The display name comes from `Uri.lastPathSegment`; the existence check happens implicitly during `read`. Avoiding extra ContentResolver round-trips keeps `open()` fast.

```bash
nix-shell --run "./gradlew test"

```

Manual on-device walkthrough:
1. Fresh install: `adb shell pm clear dev.orgdroid` (clears SharedPreferences + URI permissions). Launch → "No file open. Tap Open."
2. Open `test.org`. Background app, kill via `adb shell am force-stop dev.orgdroid`. Relaunch → `test.org` auto-opens.
3. Tap the new Close icon in the top bar → "Recent files" list with `test.org` as the only entry.
4. Tap the entry → `test.org` opens.
5. Tap Open → SAF picker → pick a second file. Now in the second file.
6. Tap Close → "Recent files" list has both files, second one at top.
7. Long-press the first entry → "Remove from recents" → it disappears.
8. Edit a title in the outline. The save bar shows "•". Tap Close → "Discard unsaved changes?" dialog. Tap Cancel → still in the file with the unsaved edit. Tap Close again → Discard → recents list.
9. Re-open the file from recents; confirm the edit was indeed discarded.
10. Open eleven distinct files (you can `adb push` synthetic `.org` files into Documents). The list should hold 10 entries; the first-opened drops off.
11. From a host shell, delete one of the recent files (or revoke its grant). Tap that entry → error banner appears above the list ("Error: …"). Long-press → Remove.
12. Rotate while on the recents list → list persists.
13. Save a file, observe its timestamp bumps (via re-opening recents — it should be on top).

When all acceptance criteria pass, M7 is done.

## What M7 explicitly does NOT do

- A settings screen to disable auto-open. Always on.
- A "Clear all recents" bulk action.
- Sync recents across devices.
- File previews, sizes, timestamps in the list.
- Pinning / favorites.
- Reissuing SAF permission for a revoked URI.
- File rename detection.
- Persisting per-file UI state (collapse, scroll, zoom).
- A Robolectric-backed `RecentFilesStore` test. Only the codec is unit-tested.
- A confirmation dialog for `removeRecent`. One-tap removal; the user can always reopen the file via SAF if they regret it.
- "Forget last open" toggle on the launcher icon.
- Adding `DataStore`, `kotlinx.serialization`, `Room`, `org.json`, or any other dep.
