# M4 — Structure ops: indent, outdent, zoom

## Goal
Add the next tier of Workflowy-style structural operations on top of M3's editing layer:

1. **Indent** (long-press menu): make this row a child of its previous sibling. Disabled if there's no previous sibling.
2. **Outdent** (long-press menu): make this row a sibling of its current parent. Disabled when the parent is already the top of the current view.
3. **Zoom in** (long-press menu): set this node as the focused "view root" — only its descendants are visible. Disabled if the node has no children.
4. **Zoom out** (back arrow in top bar): step up one level. The arrow only appears while focused; at the topmost focus it clears focus entirely.
5. **Breadcrumb-ish top bar**: when focused, the title shows the focused node's title (no file name); back arrow on the left.
6. **FAB is context-aware**: when zoomed in, the `+` appends a new child of the focused node; when not focused, a new top-level heading. (Same call site, different semantics.)

## What we keep from M3
- `OrgParser`, `OrgSerializer`, `Node`, `NodeId`, `FileIo`, `MainActivity`, `Theme`.
- Existing TreeOps (`findNode`, `transform`, `updateTitle`, `insertSiblingAfter`, `appendTopLevel`, `delete`, `maxNodeIdValue`).
- Inline editing, save with on-disk conflict detection, dirty indicator, ConflictDialog.

## Out of scope for M4 (don't implement, even if tempting)
- TODO chip tap to cycle state — M5.
- Editing notes / priority / tags — M5+.
- Move up / move down (without indent change) — later (probably with drag-reorder).
- Drag-reorder — later.
- Tab / Shift-Tab hardware keyboard shortcuts.
- Undo / redo.
- Persistence of focus across process death.
- Confirmation dialogs for any operation.
- Animated transitions when zooming.

### `app/src/main/kotlin/dev/orgdroid/org/TreeOps.kt` (additions)

Add the following inside the existing `object TreeOps`. Do **not** remove or rename any existing function.

```kotlin
/**
 * Find the parent of [id] and the index where [id] lives in parent.children.
 * Returns null if [id] is the root or not present.
 */
fun findParentAndIndex(root: Node, id: NodeId): Pair<Node, Int>? {
    for ((i, c) in root.children.withIndex()) {
        if (c.id == id) return root to i
        val sub = findParentAndIndex(c, id)
        if (sub != null) return sub
    }
    return null
}

/**
 * Add [delta] to [node.level] and every descendant. Clears [Node.rawHeadingLine]
 * on every node touched — the stars changed, so the serializer must reconstruct.
 */
private fun bumpLevels(node: Node, delta: Int): Node = node.copy(
    level = node.level + delta,
    rawHeadingLine = null,
    children = node.children.map { bumpLevels(it, delta) },
)

/**
 * Indent the node with [id]: detach from current parent, append as last child of
 * its previous sibling. No-op (returns input root) when there's no previous sibling.
 */
fun indent(root: Node, id: NodeId): Node {
    val (parent, idx) = findParentAndIndex(root, id) ?: return root
    if (idx == 0) return root
    val node = parent.children[idx]
    val prev = parent.children[idx - 1]
    val bumped = bumpLevels(node, +1)
    val newChildren = parent.children.toMutableList().apply {
        set(idx - 1, prev.copy(children = prev.children + bumped))
        removeAt(idx)
    }
    return transform(root, parent.id) { it.copy(children = newChildren) }
}

/**
 * Outdent the node with [id]: detach from current parent, insert as the next
 * sibling of its parent. No-op when the parent IS the ceiling — either the
 * synthetic root (no focus) or the [focusedRoot] (can't escape the focused view).
 */
fun outdent(root: Node, id: NodeId, focusedRoot: NodeId? = null): Node {
    val (parent, idx) = findParentAndIndex(root, id) ?: return root
    val ceiling = focusedRoot ?: root.id
    if (parent.id == ceiling) return root
    val (grandparent, parentIdx) = findParentAndIndex(root, parent.id) ?: return root
    val node = parent.children[idx]
    val newParent = parent.copy(
        children = parent.children.toMutableList().apply { removeAt(idx) }
    )
    val bumped = bumpLevels(node, -1)
    val newGrandChildren = grandparent.children.toMutableList().apply {
        set(parentIdx, newParent)
        add(parentIdx + 1, bumped)
    }
    return transform(root, grandparent.id) { it.copy(children = newGrandChildren) }
}
```

Notes for the implementer:
- `findParentAndIndex` recurses depth-first; the first match wins. NodeIds are unique so there's no ambiguity.
- `bumpLevels` is `private` — only `indent`/`outdent` use it.
- The order in `indent`'s `apply` block matters: `set` first, `removeAt` second. If you remove first, the index `idx - 1` shifts. The code above is correct as written.
- `transform` already handles the synthetic-root case (when `parent.id == root.id` in `indent`, or `grandparent.id == root.id` in `outdent`). No special-casing needed.

```kotlin
private fun flatten(
    root: Node?,
    collapsed: Set<NodeId>,
    focusedRoot: NodeId?,
): List<RenderItem> {
    if (root == null) return emptyList()
    val startNode = if (focusedRoot != null) dev.orgdroid.org.TreeOps.findNode(root, focusedRoot) else root
    if (startNode == null) return emptyList()

    val out = mutableListOf<RenderItem>()
    fun visit(parent: Node, node: Node, indexInParent: Int, depth: Int) {
        val isCollapsed = node.id in collapsed
        if (node.id != startNode.id) {
            out.add(
                RenderItem(
                    node = node,
                    depth = depth,
                    hasChildren = node.children.isNotEmpty(),
                    isCollapsed = isCollapsed,
                    canIndent = indexInParent > 0,
                    canOutdent = parent.id != startNode.id,
                )
            )
        }
        if (!isCollapsed) {
            val childDepth = if (node.id == startNode.id) 0 else depth + 1
            for ((i, c) in node.children.withIndex()) visit(node, c, i, childDepth)
        }
    }
    // Enter with node == startNode (skipped from emission); children are visited with depth=0.
    visit(startNode, startNode, -1, 0)
    return out
}
```

## Pitfalls (read before coding)

1. **Level changes invalidate `rawHeadingLine`.** `bumpLevels` clears it on every node it touches — otherwise the serializer would re-emit the original star count and your indent/outdent silently won't persist on save.
2. **`bumpLevels` is recursive.** When you indent a subtree, every descendant's `level` increases by 1 too. Same for outdent. The `indent/outdentBumpsDescendantLevels` tests catch this.
3. **Indent order: replace previous sibling FIRST, then remove.** If you call `removeAt(idx)` first, the index `idx - 1` shifts to point at a different element. The code in this spec is written in the safe order — preserve it.
4. **`findParentAndIndex` returns null for the synthetic root.** Callers must short-circuit: `indent`/`outdent` return the input root unchanged when the lookup fails; `zoomOut` treats null parent as "clear focus".
5. **Outdent ceiling.** Without focus, the ceiling is the synthetic root. With focus, the ceiling is the focused node. A row whose parent IS the ceiling cannot outdent. `TreeOps.outdent` enforces this; the UI mirrors it via `canOutdent`.
6. **`flatten` must skip the start node from emission.** When zoomed in, the focused node sits in the top bar — it's not a row. The recursion enters with `node == startNode` and the `if (node.id != startNode.id)` guard suppresses emission for it. The child loop still runs.
7. **`canIndent` = `indexInParent > 0`. `canOutdent` = `parent.id != startNode.id`.** Compute both inside `flatten` while you already have the parent and index in scope. Don't try to derive them later from `RenderItem` alone — by then you've lost the parent.
8. **The FAB's `appendInScope` MUST respect focus.** Skipping the focus check produces the surprising behaviour of "FAB inserts a top-level heading that's invisible in the current zoomed view."
9. **Delete-invalidating-focus.** If the user deletes the focused node OR any ancestor of it, the focused node disappears. `delete` uses `TreeOps.findNode(newRoot, focusedRoot)` to validate the focus survives.
10. **Open resets focus implicitly.** `open(uri)` constructs `OutlineState(uri = …, fileName = …, root = …, originalText = …)` — every other field defaults, including `focusedRoot = null`. No code change needed in `open`.
11. **`commitEditInternal` prefix on every new mutator.** `indent`, `outdent`, `zoomIn`, `zoomOut`, `appendInScope` all call `commitEditInternal` first. Skipping it loses in-progress edits when the user reaches into the menu while a text field has uncommitted changes.
12. **`Icons.Filled.ArrowBack` is in `material-icons-core`** — same package as `Add`, already pulled transitively. Don't add `material-icons-extended` (huge).
13. **Don't conflate `level` and `depth`.** `Node.level` is the org-mode star count (1, 2, 3, …). `RenderItem.depth` is the UI indentation from the current view's top. When zoomed on a level-3 node, its children get `depth = 0` while their `level` stays 4. The indent padding is computed from `depth`, the heading line from `level`.
14. **Dirty indicator hidden while zoomed.** The `•` is a file-level concept; showing it next to a focused node's title is visually confusing. The provided code hides it when `focusedRoot != null`. The dirty state itself is unchanged — Save remains enabled.
15. **`AlertDialog` and `DropdownMenu` placement (carried over from M3).** ConflictDialog stays at the top of `OutlineScreen` body. DropdownMenu stays inside the row (closes when the row scrolls out — acceptable; the menu's scrim blocks scroll in practice).
